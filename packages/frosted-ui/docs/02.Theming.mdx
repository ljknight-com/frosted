# Theming

All theming happens through the `Theme` component and CSS variables.

```tsx
import { Theme } from '@aussieljk/frosted';

<Theme accentColor="blue" grayColor="gray" appearance="inherit">
  <App />
</Theme>;
```

Configuration is managed and applied via `<Theme />`, including the semantic colors:

```tsx
<Theme
  // Sets Light or Dark mode
  appearance="dark"
  // Neutral color of the UI
  grayColor="gray"
  // Accent color (mainly used as a default color for interactive elements)
  accentColor="blue"
  // Semantic colors
  infoColor="sky"
  successColor="green"
  warningColor="yellow"
  dangerColor="red"
>
  {children}
</Theme>
```

## Accent colors

`accentColor` selects the color scale used by interactive components (buttons, switches,
checkboxes, badges, …). The scales are the Tailwind CSS v4 palettes — each one extended with an
extra light shade (step `10`) to form the 12-step scale components consume:

`red`, `orange`, `amber`, `yellow`, `lime`, `green`, `emerald`, `teal`, `cyan`, `sky`, `blue`,
`indigo`, `violet`, `purple`, `fuchsia`, `pink`, `rose`, and the neutrals `slate`, `gray`,
`zinc`, `neutral`, `stone`.

```tsx
<Theme accentColor="emerald">…</Theme>
```

Most components also accept a per-instance `color` prop that overrides the theme accent:

```tsx
<Button color="rose">Delete</Button>
<Badge color="emerald">New</Badge>
```

## Custom palettes

For brand colors not covered by the Tailwind palettes, generate a custom 12-step
scale (light + dark + alpha variants) with the palette helpers:

```tsx
import { createPaletteCss, createPaletteFromColor } from '@aussieljk/frosted';

// 1. Expand a single brand color into a full Tailwind-style 50–950 palette…
const palette = createPaletteFromColor('#5b3df5');

// 2. …then generate self-contained CSS for it. Inject it once (a CSS file or <style> tag).
const css = createPaletteCss('my-brand', palette);
```

If you already have a full Tailwind-style palette (stops 50–950), pass it to `createPaletteCss`
directly. Once the CSS is injected, the name works everywhere a scale name does:

```tsx
<Theme accentColor={'my-brand' as never}>…</Theme>
// or scope it to a subtree:
<div data-accent-color="my-brand">…</div>
// or use the raw tokens:
// var(--my-brand-9), var(--my-brand-a3), …
```

Pass `{ gray: true }` as the third argument to also emit the mapping that lets the palette be
used as the Theme `grayColor`.

## Gray scale

`grayColor` selects the neutral scale: `slate`, `gray`, `zinc`, `neutral`, `stone`, or `auto`
(picks the gray that matches your accent).

## Light and dark

- `appearance="inherit"` (recommended): follows a `light` or `dark` class on an ancestor
  element, so it integrates with any theme switcher.
- `appearance="light" | "dark"`: forces a mode and keeps the document class in sync.

## Fonts

@aussieljk/frosted components read typography from CSS variables, so overriding fonts is a one-liner:

```css
.frosted-ui {
  --default-font-family: 'Inter', sans-serif;
  --heading-font-family: 'Inter', sans-serif;
  --code-font-family: 'JetBrains Mono', monospace;
}
```

Other font variables: `--strong-font-family`, `--em-font-family`, `--quote-font-family`.
See **Guides → Typography** for the full type scale and self-hosting recipes.

## Scrollbars

Thin, themed scrollbars are opt-in. Add the `fui-scrollbar-thin` class to any scroll container
to get a slim scrollbar that follows the theme:

```html
<div class="fui-scrollbar-thin" style="overflow: auto">…</div>
```
