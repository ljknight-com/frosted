# What we lost migrating from Storybook to react-cosmos

Date: 2026-07-23. The migration replaced Storybook (docs site + stories) with react-cosmos
(fixtures + playground). Cosmos is a component workbench, not a docs framework, and the
decision was to **drop the docs site rather than rebuild it**. This file records exactly what
that gave up, so nothing is lost silently.

## 1. The rendered docs site — the big one

78 authored component MDX pages plus 10 guide pages (Getting Started, Theming, Color,
Typography, Layout, Breakpoints, Tailwind, Icons, Render Prop, Examples) no longer render
anywhere. The source files are archived in `docs/` (this directory) but nothing builds them;
their embedded `<Demo>` / `<PropsTable>` / addon-docs blocks are dead references.

Gone with the site:

- **Prose documentation** — usage guidance, do/don't notes, the Theming and Color system
  explanations that existed nowhere else.
- **`<PropsTable>`** — rendered prop tables (name, type, default, description) for every
  component. The underlying data still exists (`cosmos/generated/component-props.json`,
  regenerated by `bun run generate:props`) but nothing renders it for humans anymore.
- **`<Demo>` blocks** — live demos embedded in prose, with their source shown verbatim
  (via vite `?raw` imports) for copy-paste.
- **The Examples page** — the curated all-demos gallery with human titles and deliberate
  ordering (the demo registry). Demos survive as cosmos fixtures in `demos/`, but the
  curation metadata (titles, section grouping, order) is gone.
- **Docs search & sidebar taxonomy** — storybook's curated categories
  (`Controls/Button`, `Typography/Blockquote`) are replaced by a file-path tree
  (`src > components > button`).

## 2. `llms.txt` / `llms-full.txt`

The machine-readable docs (`https://frosted.localhost/llms.txt`, `/llms-full.txt`) were
generated from the MDX with prop tables and demo sources inlined. The generator
(`scripts/generate-llms.ts`) consumed the MDX and the storybook public dir, so it was deleted
with them. **There is no longer any LLM-consumable documentation artifact for the package.**
Restoring this would mean writing a new generator over the archived MDX (minus the storybook
block expansion) or over the fixtures.

## 3. Controls / playground fidelity

The old setup auto-backfilled storybook controls for _every_ story from the generated prop
data (`enhance-arg-types.ts`). The cosmos replacement (`useComponentControls` in
`cosmos/controls.ts`) is close but narrower:

- Controls exist **only on each component's `Default` fixture**; variant fixtures are static.
- Composition components (Dialog, Drawer, Popover, Sheet, most overlay components) got **no
  controls at all** — their Default stories render Root+Trigger+Content compositions where
  single-component controls don't apply.
- Number props without a documented default aren't controllable (cosmos inputs need a
  concrete initial value).
- Optional enum props use an `'(unset)'` sentinel instead of storybook's genuine
  undefined state.
- No color/date control matchers, and no **actions logging** (storybook auto-logged
  `on*` handlers to the Actions panel; `fn()` spies from `storybook/test` were replaced
  with `() => {}`).

## 4. Autodocs

Components without an authored MDX page had storybook-generated docs pages (prop table +
story gallery). Nothing replaces those.

## 5. Docs-page screenshots

`bun run screenshot` used to capture docs pages (the Examples page sections) in addition to
stories. It now captures fixtures only. Demo coverage is equivalent (each demo is a fixture),
but rendered-docs-page screenshots are gone, and **screenshot filenames changed** (fixture
tree paths instead of storybook story ids) — any external diff baselines are invalidated.

## 6. Deployment target

`vercel.json` deployed the full docs site (`storybook-static`). It now deploys the cosmos
export — a component browser with no prose, no prop tables, no llms.txt. The
"deploy the docs publicly" idea from the improvement backlog (archived here as
`STORYBOOK-IMPROVEMENTS.md`) mostly no longer applies.

## 7. Storybook ecosystem access

Off the table until/unless something replaces it: addon-a11y, interaction tests
(`play` functions — none existed, so nothing concrete was lost), viewport/device toolbars,
Chromatic-style visual regression integrations, and the broader addon ecosystem.

## 8. Smaller casualties

- **Story-name identity**: storybook ids (`controls-button--default`) were stable public
  anchors (deep links); cosmos fixture URLs encode file paths + fixture names instead, so
  moving a file changes its URL.
- **Per-story theme pinning** via `parameters.theme` (the mechanism existed; no story
  actually used it).
- **`new:component` scaffolding** no longer generates a docs page skeleton — only a fixture
  and a demo.
- **The `!autodocs`/MDX pairing convention** and its sharp edge are moot (one less footgun,
  but also one less docs pathway).

## What survived, for the record

- All 79 story suites live on as colocated fixture files (`src/components/*/*.stories.tsx`
  → `*.fixture.tsx`), all 80 demos as `demos/*.fixture.tsx`, theme/accent/gray switching as
  fixture selects on every fixture (decorator), prop extraction (`tools/props-gen`), the
  screenshot pipeline + contact sheet, portless dev URLs, and the static export for fast
  screenshot sweeps.
- The MDX itself is archived here and can seed a future docs site (Astro/Starlight or a
  bespoke Vite MDX app) if one is ever wanted again.
